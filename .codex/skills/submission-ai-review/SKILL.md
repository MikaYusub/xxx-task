# submission-ai-review

Use this skill before a final commit, pull request, or submission review.

## Checklist

- Score requirement coverage.
- Check structural clarity of README, DESIGN, PLAN, and code layout.
- Check failure-mode handling.
- Check production reasoning.
- Check test and verification evidence.
- Check consistency between code, README, DESIGN, PLAN, and AGENTS guidance.
- Check ignored files and generated artifacts are not staged.

## Scoring

Use this scale:

- 5: clear, complete, and production-aware.
- 4: complete with small clarity gaps.
- 3: works but has design or documentation weaknesses.
- 2: misses important requirements.
- 1: not submission-ready.

Treat documentation gaps that weaken the take-home score as P1 findings, not polish.
