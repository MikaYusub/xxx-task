# Documentation Guidance

This folder is for documentation-specific rules. Root documentation files are also governed by the
root `AGENTS.md`.

## Rules

- Keep local MVP and production redesign separate.
- README is for setup and API usage.
- DESIGN is for architecture, trade-offs, scaling, and deployment.
- PLAN is for the current implementation plan.
- Avoid vague claims such as "scalable" or "resilient" without naming the mechanism.
- Treat missing assignment coverage as a P1 issue.
- Treat README/DESIGN/PLAN inconsistencies as a P1 issue.
